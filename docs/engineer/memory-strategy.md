# 長期記憶の活用戦略

## 目的

セッション間で知識を継続し、過去の設計判断・ユーザーの好み・バグ解決策を次のセッションで活用する。

## 記憶のライフサイクル

```
保存（Input）        →  蓄積（Storage）  →  検索（Output）       →  活用
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
manual: memory_save     PostgreSQL        memory_search          コンテキスト注入
auto: PostSessionEnd    embedding 384dim  hybrid search          設計判断の参照
                        dedup >= 0.95     RRF + time decay       過去のバグ回避
```

## 保存戦略 — 何を保存するか

### 必ず保存するもの

| カテゴリ | タグ | 例 |
|---------|------|-----|
| 設計判断 | `design` | 「RRFのk=60にした理由」「huskyに移行した理由」 |
| ユーザーの好み | `preference` | 「日本語でドキュメント」「自然な技術選定を重視」 |
| バグ原因・解決策 | `bug` | 「flakyテストの原因はタイムスタンプ同一」 |
| プロジェクトルール | `rule` | 「CLAUDE.mdは目次+リンク」「二重管理しない」 |
| アーキテクチャ知識 | `architecture` | 「coreは依存ゼロ」「hook はdocker execで起動」 |

### 保存しないもの

- 一般的な技術知識（公式ドキュメントに書いてあること）
- 一時的な作業状態（「今○○をしている」）
- コードそのもの（gitで管理されている）
- 自動生成可能な情報（What系ドキュメント）

## 検索戦略 — いつ・何を検索するか

### セッション開始時（必須）

以下のクエリで関連記憶を検索し、コンテキストを把握する：

1. **プロジェクト名で検索** — `memory_search("claude-memory")`
2. **今日の作業内容で検索** — `memory_search("検索アルゴリズム改善")` 等
3. **ユーザーの好みを検索** — `memory_search("preference", tags: ["preference"])`

### 作業中（状況に応じて）

- **設計判断時** — 過去に同様の判断をしていないか検索
- **バグ遭遇時** — 同じバグの解決策がないか検索
- **新技術導入時** — 過去の技術選定理由を検索

### セッション終了時（自動）

PostSessionEndフックが自動で会話をQ&Aチャンクに分割して保存。重複チェック（cosine >= 0.95）済み。

## タグ体系

| タグ | 用途 | 検索場面 |
|------|------|---------|
| `design-decision` | 設計判断とその理由 | 新しい設計判断時に過去を参照 |
| `preference` | ユーザーの好み・スタイル | セッション開始時に毎回参照 |
| `bug` | バグ原因と解決策 | バグ遭遇時 |
| `rule` | プロジェクトルール | ルール確認時 |
| `architecture` | アーキテクチャ知識 | 設計作業時 |
| `process` | 開発プロセスの知見 | ワークフロー改善時 |
| `documentation-strategy` | ドキュメント方針 | ドキュメント作成時 |

## ADRとの連携

設計判断は **ADR（docs/adr/）と長期記憶の両方に保存** する。

- **ADR** — 構造化された記録。PRレビューで確認。Gitで履歴管理
- **長期記憶** — 検索可能な記憶。セッション開始時にAIが自動参照

同じ判断を2箇所に保存するが、役割が異なるため二重管理ではない：
- ADRは「人間が読む構造化ドキュメント」
- 長期記憶は「AIが検索するためのインデックス」

## CLAUDE.mdとの連携

CLAUDE.mdの記憶ルールに以下を明記する：

```markdown
## 記憶ルール
- セッション開始時に memory_search で **必ず** 検索する（省略しない）
- 設計判断があったらADR + memory_save の両方に保存する
- tags を必ず付ける（検索精度に直結）
```

## メンテナンス

- **memory_cleanup** — 90日以上アクセスされていない記憶を定期的にクリーンアップ（dry-run で確認後に実行）
- **memory_stats** — 記憶の総数・manual/auto比率を定期確認
- **memory_export** — 定期バックアップ
